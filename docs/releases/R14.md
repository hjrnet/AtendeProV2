# R14 — Alinhamento funcional e completude do produto

## Objetivo
Tornar o estado de prontidão do AtendePro explícito e funcional, corrigindo lacunas administrativas e conectando operações pendentes para que o produto seja realmente "completo".

## Frentes de arquitetura e execução

- Garantir governança honesta de release/finalização (sem `CONCLUIDA` para fluxos futuros).
- Implementar base operacional adicional para verticais não-finalizadas (Biomed/Fisio) com escopo claro de "primeiro MVP operacional".
- Concluir o mobile para consumo real de backend (auth + endpoints reais dos fluxos principais).
- Consolidar análises de falta funcional por release/vertical e atualizar documentação para reduzir risco de execução no escuro.

## Tasks

- TASK-R14-001 — Correção de governança de conclusão e contrato de status de funcionalidades futuras.
- TASK-R14-002 — Inventário técnico de lacunas e plano de estabilização de verticais futuras (Biomed/Fisio/Psico/Fono/Farmácia/Odonto).
- TASK-R14-003 — Conectar autenticação e shell mobile ao backend real (login, sessão e validação de tenant/empresa).
- TASK-R14-004 — Conectar os fluxos mobile (cliente/profissional) a dados reais de agenda, pacientes e documentos.
