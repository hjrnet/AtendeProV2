# R30 - Pagamentos sandbox com arquitetura hexagonal

Status: CONCLUIDA

## Objetivo

Implementar o modulo de pagamentos em sandbox, usando o contrato tecnico definido na R29, sem acionar cobrancas reais e mantendo o gateway externo isolado por portas e adapters.

## Tasks concluidas

- TASK-R30-001: Criar modulo pagamento sandbox.

## Evidencias

- Release criada como continuidade governada da R29.
- Provedor e contrato tecnico base: `docs/architecture/PAGAMENTOS_GATEWAY_R29.md`.
- Variaveis de ambiente previstas: `.env.example` e `docs/deploy/ENVIRONMENT.md`.
- Modulo backend `pagamento` criado com dominio, use cases, ports, controller e adapter JDBC.
- Migration `0043-create-pagamentos-sandbox-r30.yaml` criada para assinaturas de pagamento, cobrancas e eventos de gateway.
- Endpoint `POST /api/admin-saas/pagamentos/checkout/sandbox` prepara checkout sandbox sem cobranca real.
- Endpoint `POST /api/admin-saas/pagamentos/webhooks/asaas` registra webhooks em sandbox com token, idempotencia e reconciliacao local.
- Testes unitarios adicionados para dominio e service de pagamento.
