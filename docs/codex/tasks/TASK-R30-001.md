# TASK-R30-001 - Criar modulo pagamento sandbox

Status: CONCLUIDA

## Objetivo

Criar a base backend hexagonal do modulo de pagamentos em modo sandbox, usando o contrato tecnico da R29 e sem gerar cobranca real.

## Escopo previsto

- Criar dominio, commands/results, use cases e portas do modulo `pagamento`.
- Criar adapter inicial sandbox/Asaas sem credenciais reais versionadas.
- Criar migrations minimas para assinatura, pagamento e eventos de webhook.
- Expor contratos internos seguros para checkout/assinatura em sandbox.
- Registrar auditoria operacional dos eventos sensiveis.

## Fora do escopo

- Cobranca real em producao.
- Credenciais reais em arquivos do projeto.
- Webhook publico de producao sem decisao humana explicita.

## Criterios de aceite

- Arquitetura hexagonal preservada.
- Testes automatizados cobrindo regras principais.
- Auditoria documental strict aprovada.
- Docker/local validado ou justificativa registrada.

## Resultado

- Modulo `pagamento` criado no backend com dominio, commands/results, use cases, portas e adapter JDBC.
- Checkout sandbox exposto em `/api/admin-saas/pagamentos/checkout/sandbox`.
- Webhook Asaas sandbox exposto em `/api/admin-saas/pagamentos/webhooks/asaas`, com token, idempotencia e reconciliacao local.
- Migration `0043-create-pagamentos-sandbox-r30.yaml` adicionada ao changelog master.
- Eventos sensiveis registram auditoria Admin SaaS.
- Producao permanece bloqueada na R30.
