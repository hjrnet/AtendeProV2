# TASK-R32-001 - Implementar observabilidade e reconciliação de pagamentos sandbox

Status: CONCLUIDA

## Objetivo

Implementar visão operacional de observabilidade e reconciliação para pagamentos sandbox no Admin SaaS, com indicadores, divergências e ação guiada para recuperação no ambiente de testes.

## Escopo previsto

- Criar no backend o contrato de consulta observacional de pagamentos sandbox (indicadores + divergências).
- Implementar query de indicadores, classificação por evento e severidade e persistência de retorno por DTO.
- Expor endpoint protegido em Admin SaaS para leitura operacional da base sandbox.
- Ajustar o painel do Admin SaaS (`Pagamentos R31`) com filtros, cards de indicadores e lista de divergências.
- Disponibilizar ação de reconciliação para divergências críticas por disparo de webhook local.

## Fora do escopo

- Pagamento em produção.
- Cobrança real.
- Mudança de regra financeira de assinatura e faturamento definitivo.
- Novas integrações externas não previstas no contrato R29/R30.

## Critérios de aceite

- Backend e web compilam sem regressões.
- Testes obrigatórios executados.
- Documentação oficial de release atualizada.
- Arquitetura respeita camada hexagonal e interfaces de input/output.
- Risco de divergência tratável e observável via filtro/dashboard no Admin SaaS.
