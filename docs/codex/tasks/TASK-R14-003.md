# TASK-R14-003 — Conectar autenticação e shell mobile ao backend real

## Release
R14 — Alinhamento funcional e completude do produto

## Complexidade
MÉDIA

## Objetivo
Substituir autenticação/dados de demonstração do mobile por integração real com backend e contexto de tenant quando aplicável.

## Contexto
Atualmente o app mobile funciona como estrutura de experiência, com API mínima (health) e utilitários de autenticação simples.

## Escopo permitido
- Implementar login real via endpoint `/api/auth/login`.
- Persistir token de sessão para chamadas autenticadas no app.
- Integrar resolução de empresa/tenant para as chamadas seguintes.
- Mostrar erro explícito no app em falha de credenciais/ambiente/tenant.

## Fora de escopo
- Implementar todos os fluxos funcionais de negócio do mobile (agenda/pacientes/documentos) desta task.
- Mudanças de identidade visual extensivas.

## Regras técnicas
- `mobile/app` com hook/páginas já existente.
- Manter mensagens e labels em português claro.
- Sem push.

## Critérios de aceite
- Login mobile autentica no backend real.
- Sessão persistida no app (Storage seguro para runtime).
- Requisição autenticada para endpoint de validação básica retorna tenant/empresa/contexto.
- Estados de loading/erro/fallback implementados.

## Validação
- Testes locais manuais no mobile (login inválido/válido, timeout, token inválido).
- Validação rápida de fluxo `demo local`.
