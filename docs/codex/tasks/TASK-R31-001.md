# TASK-R31-001 - Integrar cockpit Admin SaaS ao pagamento sandbox

Status: CONCLUIDA

## Objetivo

Expor consulta operacional de pagamentos sandbox no backend e conectar o cockpit Admin SaaS web aos endpoints de checkout e webhook criados na R30.

## Escopo previsto

- Criar consulta backend para pagamentos sandbox do Admin SaaS.
- Tipar API client web para checkout sandbox, webhook Asaas sandbox e listagem de pagamentos.
- Adicionar painel R31 no cockpit Admin SaaS.
- Permitir simular eventos `PAYMENT_RECEIVED`, `PAYMENT_OVERDUE` e `PAYMENT_DELETED` sem cobranca real.
- Atualizar governanca e observabilidade da release.

## Fora do escopo

- Producao Asaas real.
- Credenciais reais.
- Checkout publico.
- Mudanca de regra financeira de ativacao de assinatura SaaS.

## Criterios de aceite

- Backend e web compilam.
- Testes automatizados passam.
- Docker/local validado ou justificativa registrada.
- Auditoria documental strict aprovada.

## Resultado

- Consulta backend `GET /api/admin-saas/pagamentos/assinaturas` adicionada para pagamentos sandbox.
- Client web do Admin SaaS passou a consumir checkout, webhook e listagem de pagamentos.
- Nova aba `Pagamentos R31` adicionada ao cockpit Admin SaaS.
- Simulacao segura de `PAYMENT_RECEIVED`, `PAYMENT_OVERDUE` e `PAYMENT_DELETED` criada no web app.
- Producao Asaas e credenciais reais permanecem fora do escopo.
