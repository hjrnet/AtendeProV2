# TASK-R14-004 — Conectar fluxos mobile a dados reais

## Release
R14 — Alinhamento funcional e completude do produto

## Complexidade
ALTA

## Objetivo
Conectar mobile a endpoints reais para os fluxos iniciais com estado de UX útil (loading, vazio, erro).

## Contexto
Mobile apresenta dados mockados para agenda/clientes/documentos. Esta task transforma o MVP de apresentação em fluxo operacional inicial.

## Escopo permitido
- `Agenda`, `clientes/pacientes` e `documentos` do app mobile consumindo APIs reais.
- Telas `cliente/*` e `profissional/*` com estados e mensagens claros.
- Reuso de tipagem e contratos já existentes no web client quando possível.

## Fora de escopo
- Reescrever arquitetura do app do zero.
- Implementar funcionalidades finais de chat/diário/mensagens fora de prioridade do slice inicial.

## Regras técnicas
- Seguir contratos de API existentes.
- Manter estados de UI previsíveis (loading/vazio/erro).
- Sem push.

## Critérios de aceite
- App mobile exibe dados reais para agenda/documentos/clientes com fallback quando não houver dados.
- Não mais depende de arquivo `features/*/*.ts` com dados totalmente estáticos para os fluxos conectados.
- Teste manual em telas prioritárias de cliente e profissional.
- Relatório de cobertura funcional da integração atualizado.

## Validação
- Validação manual mobile em fluxo: login -> listagem -> detalhes -> recarregar.
- Teste de ausência de conexão e resposta vazia.
